٤) سكربت release.bat
أنشأته الآن في جذر المشروع. ينفذ بنقرة مزدوجة:

يكتشف JDK 21
يقرأ رقم النسخة من AppMetadata.kt تلقائيا
يشغل الاختبارات (يلغي البناء لو فشلت)
يبني MSI
يفتح مجلد MSI في المستكشف
يفتح صفحة GitHub Releases في المتصفح
الجزء اليدوي الوحيد: سحب MSI إلى صفحة GitHub، وكتابة وصف الإصدار.

⏭️ ما تتوقعه: دورة الإصدار الكاملة
كل مرة تريد إصدار نسخة جديدة، الخطوات بترتيبها:

قبل الإصدار (أنت أو معي)
أضيف ميزة جديدة أو أصلح خطأ
أرفع رقم النسخة في ملفين (سطر واحد لكل ملف)
أكتب ما تغير في CHANGELOG.md
وقت الإصدار (أنت بنفسك)
انقر مزدوجا على release.bat — ينتظر 5 دقائق
صفحتان تفتحان: مجلد MSI + GitHub Releases
تسحب MSI إلى الصفحة، تكتب اسم الإصدار v0.6.0، تنسخ الملاحظات من CHANGELOG، تنقر "Publish"
بعد الإصدار (دقيقة واحدة)
تعدل version.json على موقعك:
{ "version": "0.6.0", "url": "<رابط GitHub>" }
النتيجة: كل المستخدمين الذين يفتحون البرنامج يرون إشعار "نسخة جديدة متاحة".
